# Functional QA Pass

Execute a functional QA pass on the current screen or feature: <describe>.

1. Read and apply `dev-obsidian/_Compartilhado/Skills/project-test-orientation/SKILL.md` to identify the correct stack, container, port, and health endpoint.
2. Read and apply `dev-obsidian/_Compartilhado/Skills/qa-functional-review/SKILL.md`.
3. Exercise the real screen in a browser against the running development server; do not validate only through source-code reading.
4. Record every confirmed gap that is not fixed in the same change set in `<project>/06_Pendencias/`.
5. Report the exercised flows, findings, fixes applied, pending items created, and coverage limits.
